## 方法(忠实于 Ouro)

- **循环 = 全栈权重共享递归**(非 latent-only)。
- **可学习 per-step 残差门控** `_ouro_step_gate`:第 0 步是完整(无门控)预训练 forward;
  额外步是门控残差精炼 `x ← x + tanh(g_k)·(stack(x) − x)`。
- **per-step 输出监督**:每一步都 decode test 行算 cross-entropy。
- **固定 T_max = 4** 训练(Ouro 设定)。gate 用更快的 LR(1e-3),其余参数 1e-5。

实现:复用 A 的紧凑训练循环;forward 在 ouro 模式下用门控,训练时收集 per-step decode
logits 到 `self._thinking_step_logits` 供损失读取。

> **多 subagent 审查发现两个 bug,已修(见 reproduce/run_finetune_ouro.py)**:
> 1. **熵正则是空操作**:`w = softmax(−ce.detach())` 中 w 被 detach,熵项对参数梯度为零。
>    改为**均匀加权 per-step CE 的 mean**(每步等权监督,梯度干净,且不会像 softmax 那样
>    反而降权最该改的高 loss 步)。
> 2. **门控梯度陷阱**:gate 初始 0 ⇒ step 1 输出 = step 0 ⇒ `x_stack−x_in≈0` ⇒ gate 梯度
>    ≈5e-10,永远学不动。改为**小随机初始化**(×0.05)打破对称,gate 梯度恢复到 ~5.5。
>
> 修复后实测:gate 能学、per-step CE 各步分化。**但下面的效用结论方向不变**(见"修复后结果")。

## 机制发现(正面 ✅):门控 + per-step 监督**治好了 Phase 0 的坍缩**

Phase 0 里裸 ouro 全栈循环坍缩到平凡不动点(drift→0.2、预测冻死)。加门控 + per-step
监督后:
- **不再坍缩**:微调后 drift 受控(如 breast_cancer 584→2.5→2.3,wine 413→0.7),per-step
  CE 各步分化且都成了有用预测器,深度循环内部自洽稳定。

这一条是真实的、正面的:**Ouro 的稳定化配方在 TabPFN 上确实复现了**。

## 效用结论(负面 → 经补救后精确化为"无害但无益")

### 第一轮(自由微调,bug 修复后):仍净亏损

> **对照系必须选对**:`Δacc(post−pre@steps>1)` 的分母是未门控的 Phase 0 坍缩值
> (0.28–0.37),会把"post 没坍缩"显示成 +0.49/+0.68 的幻觉。唯一正确基线是
> `pre@steps=1` = 标准 TabPFN 单遍。

按正确基线(pre@1)对比绝对准确率(3 seeds,bug 修复后 `ouro_run_v2.out`):

| 数据集 | pre@1(真基线) | post 最好 | 净效果 |
|---|---|---|---|
| wine | **1.000** | 0.957 | −0.04 |
| breast_cancer | **0.971** | 0.762 | −0.21 |
| digits | **0.985** | 0.913 | −0.07 |
| phoneme | **0.906** | 0.660 | −0.25 |
| qsar-biodeg | **0.869** | 0.728 | −0.14 |

两个 bug 修复后**结论方向不变,所有 post 仍低于基线** —— 这反而是更强的结论:**负结果不是
bug 造成的假象**。且 post@1 本身就崩(非深度循环之过),指向"微调侵蚀了预训练单遍表示"。

### 第二轮(补救:冻结编码器 + 早停)→ 损害消失,假设证实 🎯

冻结输入编码器/embedder(`feature_group_embedder`、`target_embedder`、
`feature_positional_embedding_embeddings`,共 ~11K 参数),只训 block + decoder +
thinking rows + gate,并用留出验证集早停(`run_finetune_ouro_frozen.py`):

| 数据集 | 基线 pre@1 | 自由微调 post@1 | **冻结后 post@1** | std 变化 |
|---|---|---|---|---|
| wine | 1.000 | 0.95 | **1.000±0.000** | — |
| breast_cancer | 0.971 | 0.76 | **0.961±0.010** | ±0.27→**±0.01** |
| digits | 0.985 | 0.90 | **0.927±0.058** | 改善 |
| phoneme | 0.906 | 0.66 | **0.902±0.008** | ±0.21→**±0.01** |
| qsar-biodeg | 0.869 | 0.73 | **0.861±0.007** | ±0.19→**±0.01** |

**冻结编码器后,所有数据集 post@1 都从崩塌恢复到基本追平基线(差 0.000–0.008),且方差从
±0.2 量级骤降到 ±0.01。** 这从因果上证实:之前的损害**来自小数据微调侵蚀预训练编码器**,
不是循环机制。方差崩塌是铁证 —— 之前的不稳定就是编码器被随机训坏。

**但"计算换精度"仍未实现**:冻结后 post@steps≥2 基本等于 post@1(wine 全 1.0,
phoneme 0.902 vs 0.902),没有"步数越多越好"。即两个障碍都攻克后,深度循环是**无害但无益**。

---

## 根因分析(经补救实验确认)

1. **损害 = 微调侵蚀编码器**(已证实,非推测):冻结 ~11K 编码器参数即完全挡住损害,
   方差从 ±0.2 → ±0.01。这与"深度循环有害"无关 —— gate 几乎不动,破坏发生在 step 0
   的单遍表示上。
2. **门控成功防坍缩,但深度无净增益**:稳定 ≠ 有用。已饱和的表格 ICL 在加深度后没有可榨取的
   红利(呼应 Phase 0 判断)。

## 与 Branch A(COCONUT)对比 — 合并结论

| | Branch A (coconut) | Branch B (ouro) |
|---|---|---|
| 循环稳定性 | latent-only,天然不坍缩 | 全栈,靠门控防坍缩(成功) |
| 深度收益 | 微弱、不稳健 | 内部自洽但无净增益 |
| 对基线(pre@1)的影响 | 基本不破坏 | 自由微调显著损害;**冻结编码器后恢复** |
| 净判断 | 无稳健增益 | 攻克两障碍后:**无害但无益** |

**合并负结论**:在 TabPFN v2.5 这种已近饱和的表格 ICL 模型上,无论 COCONUT 式 latent 反馈
还是 Ouro 式全栈递归,**深度循环都拿不到"计算换精度"的净收益**。两个障碍已被逐一攻克并验证:
- **循环坍缩** → Ouro 门控解决;
- **微调侵蚀预训练表示** → 冻结编码器 + 早停解决(方差 ±0.2→±0.01 为铁证)。

攻克后剩下的是干净的负结果:**深度循环在此类模型上无害但无益,天花板本就有限**(呼应 Phase 0)。

## 局限与边界

- 5 个数据集偏小、偏易(单遍 acc 0.87–1.0),"近饱和"结论对更难/更大数据集未必成立;
  深度循环的潜在红利可能需要在远未饱和的任务上才显现。
- n_estimators=1(单 forward,便于干净测);集成下未测。
- gate 仅 T_max−1=3 个标量,容量极小(有意的保护设计)。
- 冻结实验证明了"损害来自编码器侵蚀",但未穷尽所有补救(如 KL 锚定、更大数据);
  结论是"无益",非"不可能"。
