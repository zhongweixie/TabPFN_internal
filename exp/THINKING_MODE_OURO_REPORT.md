## 方法(忠实于 Ouro)

- **循环 = 全栈权重共享递归**(非 latent-only)。
- **可学习 per-step 残差门控** `_ouro_step_gate`:第 0 步是完整(无门控)预训练 forward;
  额外步是门控残差精炼 `x ← x + tanh(g_k)·(stack(x) − x)`。门控初始 0 ⇒ **T>1 在初始化时
  逐位等于 T=1**(实测 maxdiff 0.0),循环开局为近恒等映射 —— 这正是治 Phase 0 全栈坍缩的解。
- **per-step 输出监督**:每一步都 decode test 行算 cross-entropy,按 exit 分布(对 per-step
  loss 的 softmax)加权 + 熵正则(β=0.05)防止塌到单步。
- **固定 T_max = 4** 训练(Ouro 设定)。gate 用更快的 LR(1e-3),其余参数 1e-5。

实现:复用 A 的紧凑训练循环;forward 在 ouro 模式下用门控,并在训练时收集 per-step
decode logits 到 `self._thinking_step_logits` 供损失读取。

---

## 机制发现(正面 ✅):门控 + per-step 监督**治好了 Phase 0 的坍缩**

Phase 0 里裸 ouro 全栈循环坍缩到平凡不动点(drift→0.2、预测冻死)。加门控 + per-step
监督后:
- **初始化保护成立**:gate=0 时 T=4 与 T=1 预训练逐位相同(maxdiff 0.0)。
- **不再坍缩**:微调后 drift 受控(如 breast_cancer 584→2.5→2.3,wine 413→0.7),per-step
  CE 四步几乎相等(每步都成了有用预测器),深度循环内部自洽稳定。

这一条是真实的、正面的:**Ouro 的稳定化配方在 TabPFN 上确实复现了**。

---

## 效用结论(负面 ❌):微调整体**损害**了模型

> **关键:对照系必须选对。** `Δacc(post−pre)` 里 `pre@steps>1` 是**未门控的 Phase 0 坍缩值**
> (0.28–0.37),用它当分母会把"post 没坍缩"显示成巨大"提升"(+0.49/+0.68),这是**幻觉**。
> 唯一正确的基线是 `pre@steps=1` = 标准 TabPFN 单遍。

按正确基线(pre@1)对比绝对准确率(3 seeds, mean±std):

| 数据集 | pre@1(真基线) | post 最好(任意深度) | 净效果 |
|---|---|---|---|
| wine | **1.000±0.000** | 0.951±0.070 | **−0.05** |
| breast_cancer | **0.971±0.000** | 0.715±0.246 | **−0.26** |
| digits | **0.985±0.002** | 0.821±0.047 | **−0.16** |
| phoneme | **0.906±0.005** | 0.631±0.233 | **−0.28** |
| qsar-biodeg | **0.869±0.008** | 0.731±0.195 | **−0.14** |

**没有任何数据集的 post 超过基线。** post@1 普遍崩塌(0.97→0.72),且 std 很大(不稳)。

---

## 根因分析

1. **冒烟测试骗了人**:单 seed + 4 epoch 时 wine post 全 1.0,看着完美。全量 3-seed/12-epoch
   后 post@1 普遍崩 —— 典型的小数据过拟合。

2. **per-step 监督把梯度灌进全部 250 个参数(含编码器)**,12 epoch 小数据微调侵蚀了预训练
   原本优秀的单遍表示。模型学会了"让 4 步彼此一致",但收敛到的那个不动点**比预训练单遍更差**。

3. **门控驯化了循环、却没驯化微调本身**:drift 不坍缩(机制成功),但端到端效用为负
   (微调有害)。问题从"循环能否稳定"转移到了"微调会不会毁掉预训练能力"。

---

## 与 Branch A(COCONUT)对比 — 合并结论

| | Branch A (coconut) | Branch B (ouro) |
|---|---|---|
| 循环稳定性 | latent-only,天然不坍缩 | 全栈,靠门控防坍缩(成功) |
| 深度收益 | 微弱、不稳健 | 内部自洽但整体微调有害 |
| 对基线(pre@1)的影响 | 基本不破坏 | **显著损害(−0.05 ~ −0.28)** |
| 净判断 | 无稳健增益 | **弊大于利** |

**合并负结论**:在 TabPFN v2.5 这种已近饱和的表格 ICL 模型上,无论 COCONUT 式 latent 反馈
还是 Ouro 式全栈递归,**深度循环微调都拿不到"计算换精度"的净收益**。Ouro 的门控成功解决了
"循环坍缩",但暴露了更根本的障碍:**在小数据上微调会侵蚀预训练的优秀单遍表示,得不偿失。**

## 局限与可能的补救(若要继续)

- 微调有害的直接嫌疑是**过拟合 + 灾难性遗忘**。可试:(a) 冻结编码器/embedder,只训 block
  + gate;(b) 大幅降 LR / 早停;(c) 加 KL 锚定到预训练输出(防遗忘);(d) 用更多、更大的
  训练数据集而非 5 个小集。
- 但即便补救成功,Phase 0 已表明表格 ICL 单遍近饱和,深度循环的天花板可能本就有限。
- n_estimators=1;集成下未测。gate 仅 T_max−1=3 个标量,容量极小(这是有意的保护设计)。
