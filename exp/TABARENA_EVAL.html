The two-bug story (why earlier numbers were wrong)

This eval was corrupted twice by the same class of bug — a fragile global LOOP_K controlling how many times each model loops:
  1. Bug 1 (benchmark eval): build_model only set the global when loop_k>1 and never reset it → after loading c2 (LOOP_K=2), c3 ran with 2 loops → c3 falsely "collapsed" to 0.495.
  2. Bug 2 (this TabArena eval): all 3 models built up front; the patched forward read the global at call time, which was left at c3's value (1) → c2 ran as loop_k=1 (but its weights were trained for loop_k=2) → c2 falsely "collapsed" (mean AUC 0.78). On splice: buggy 0.833 vs correct 0.960.
Root-cause fix: loop_k is now bound per model instance onto each LayerStack (self._loop_k) via set_loop_on_model(); the patched forward reads the instance attribute. Verified contamination-proof: interleaving c1/c2/c3 inference in any order, and even poisoning the old global to 99, leaves every model's output identical. No global state left to corrupt.
