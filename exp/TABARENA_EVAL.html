⚠ The metrics disagree — read carefully. c2 (looped) has the highest mean AUC but the worst mean rank (2.97) and is "best" on only 5/38 datasets, vs CatBoost's 11. How can both be true? Because c2 wins big but loses often: vs c3 it wins 18 datasets by avg +0.017 AUC, and loses 20 by only −0.008. Its few large wins pull the mean to #1, but its many small losses sink its rank and normalized score. So "c2 is best" is true only under the mean-AUC lens; by per-dataset consistency, CatBoost and c3 are at least as good. The honest one-liner: c2 ≈ c3 ≈ CatBoost, statistically tied, each with a different shape — c2 is spikier (big wins, small losses), CatBoost/c3 are steadier.

  • Metric: ROC-AUC, OvR-macro for multiclass (TabPFN-3 uses OvR; TACO uses OvO — minor difference). This is the headline classification metric in both papers — unlike our earlier ad-hoc eval which used accuracy (closer to VIP-COp's balanced-accuracy protocol).
  • Splits: official OpenML task train/test split, fold 0.
  • Baselines: CatBoost + XGBoost (defaults) for external context.

So absolute numbers are NOT comparable to the TabArena leaderboard — they only support the internal c1/c2/c3-vs-GBDT comparison at this fixed budget. The two-bug story (why earlier numbers were wrong)

This eval was corrupted twice by the same class of bug — a fragile global LOOP_K controlling how many times each model loops:
  1. Bug 1 (benchmark eval): build_model only set the global when loop_k>1 and never reset it → after loading c2 (LOOP_K=2), c3 ran with 2 loops → c3 falsely "collapsed" to 0.495.
  2. Bug 2 (this TabArena eval): all 3 models built up front; the patched forward read the global at call time, which was left at c3's value (1) → c2 ran as loop_k=1 (but its weights were trained for loop_k=2) → c2 falsely "collapsed" (mean AUC 0.78). On splice: buggy 0.833 vs correct 0.960.
Root-cause fix: loop_k is now bound per model instance onto each LayerStack (self._loop_k) via set_loop_on_model(); the patched forward reads the instance attribute. Verified contamination-proof: interleaving c1/c2/c3 inference in any order, and even poisoning the old global to 99, leaves every model's output identical. No global state left to corrupt.

After fixing both bugs, the two eval families agree that c2 has the highest mean score on both metrics — but the TabArena rank/normalized view adds an important caveat: c2's lead comes from a few large wins offset by many small losses, so by per-dataset consistency it only ties c3 and CatBoost rather than beating them. Net: looping is not harmful, is the most parameter-efficient option, and is competitive with tuned CatBoost — but "looped is best" is a mean-driven, not rank-driven, statement. This is a weak/mixed positive that warrants scaling up (TACO-level batch/data, loop_k=3/4, Ouro adaptive halting, multi-fold, more seeds) to learn whether the spiky-wins pattern is real signal or noise — not yet a strong claim.
